¿Qué es HR Copilot?

HR Copilot transforma la manera en que los líderes de Práxedes interactúan con los datos de talento humano.

En lugar de depender de analistas para construir dashboards estáticos, los gerentes pueden consultar a la base de datos utilizando lenguaje natural.

  • Autonomía inmediata (Self-service).
  • Gráficos generados dinámicamente.
  • Control estricto de seguridad de la información.

Del Texto al Gráfico

El usuario pregunta:
"¿Cuál es la rotación de personal en el equipo de ventas este año?"


La IA comprende, traduce a SQL, extrae los datos y dibuja el gráfico ideal de manera automática, respetando la gobernanza de datos.

Arquitectura y Componentes

Un sistema robusto compuesto por tecnologías líderes en el mercado:

UI & Dash

Aplicación web interactiva. Maneja la sesión del usuario, captura las preguntas y renderiza los componentes visuales con estilo corporativo.

Vanna AI (RAG)

Base de datos vectorial (ChromaDB) que almacena la estructura de las tablas (DDL) y enseña a la IA cómo hacer consultas precisas.

Gemini (LLM)

El cerebro traductor. Interpreta el lenguaje natural, genera el código SQL y también crea el código para las visualizaciones.

SQLite (DB)

La base de datos estructurada donde reside de forma segura la información de los colaboradores.

Interceptor RLS

La capa de seguridad (Row-Level Security). Controla qué datos puede ver el usuario según su rol antes de ejecutar el SQL.

Plotly

Librería de gráficos interactivos, configurada estrictamente con los colores y formas de Práxedes S.A.S.

Fase 1: Text-to-SQL (Vanna + Gemini)

¿Cómo aseguramos que la IA no invente columnas o tablas?

Utilizamos RAG (Retrieval-Augmented Generation) a través de Vanna AI. Cuando el usuario hace una pregunta, Vanna busca en su ChromaDB local el contexto relevante.

Gemini recibe un prompt hiper-especializado que contiene la pregunta del usuario + el esquema exacto de nuestra base de datos, garantizando 99% de precisión en la sintaxis SQL.

El Proceso Invisible

  • 1. Usuario: "¿Salarios promedio?"
  • 2. Vanna busca DDL de tabla "employees".
  • 3. Vanna envía contexto a Gemini.
  • 4. Gemini responde:
    SELECT AVG(MonthlyIncome) FROM employees

Fase 2: Seguridad e Interceptor (RLS)

El núcleo de la gobernanza de datos. Antes de que el SQL generado por Gemini llegue a la base de datos, pasa por rls.py.

Bloqueo de Columnas Sensibles

Si el usuario es "Employee Viewer", el interceptor escanea el SQL buscando columnas como MonthlyIncome.

ACCESO DENEGADO

El SQL se detiene y se registra en auditoría.

Inyección de Filtros (Row-Level Security)

Si el usuario es "Sales Manager", el interceptor reescribe el SQL dinámicamente.

... WHERE e.department_id = 1

Solo verá datos de Ventas.

Fase 3: Visualización Dinámica

Una vez que la base de datos retorna un DataFrame válido, el sistema debe graficarlo. ¿Cómo mantenemos el diseño de Práxedes?

Plotly + Gemini

Se envía una muestra de los datos y un estricto manual de reglas visuales a Gemini (PLOTLY_PROMPT).

Gemini escribe el código Python para generar el gráfico utilizando solo la paleta corporativa: Naranja, Blanco y Grises.

Fallback Inteligente (smart_chart)

Si la IA falla al generar el gráfico, la app tiene un motor de reglas interno que dibuja automáticamente el gráfico perfecto (Dona para porcentajes, Barras para categorías, KPIs para valores únicos).
